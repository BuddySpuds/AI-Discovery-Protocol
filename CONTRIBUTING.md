# Contributing to AI Discovery Protocol

Thank you for your interest in contributing to the AI Discovery Protocol! This open standard thrives on community input and collaboration.

## How to Contribute

### 1. Report Issues
- Use [GitHub Issues](https://github.com/BuddySpuds/AI-Discovery-Protocol/issues) to report bugs or suggest improvements
- Check existing issues before creating a new one
- Provide clear reproduction steps for bugs
- Include relevant URLs, examples, or code snippets

### 2. Propose Changes
- Fork the repository
- Create a feature branch (`git checkout -b feature/your-feature-name`)
- Make your changes with clear commit messages
- Submit a pull request with a detailed description

### 3. Share Implementations
We welcome implementation examples for different platforms:
- WordPress plugins
- Shopify apps
- Drupal modules
- Static site generators (Gatsby, Next.js, Hugo)
- CMS integrations (Contentful, Sanity, Strapi)

**To contribute an implementation:**
1. Create a directory in `implementations/` or `examples/`
2. Include a README.md with setup instructions
3. Provide working code examples
4. Document any dependencies

### 4. Improve Documentation
- Fix typos, clarify explanations, add examples
- Translate documentation to other languages
- Create tutorials or video guides
- Share success stories from real-world implementations

### 5. Develop Tools
We need community-built tools:
- Validation tools (Python, JavaScript, CLI)
- File generators (auto-generate ADP files from websites)
- Browser extensions (validate ADP compliance)
- Test suites (automated compliance checking)

## Code of Conduct

### Our Standards

Examples of behavior that contributes to a positive environment:
- Using welcoming and inclusive language
- Being respectful of differing viewpoints
- Gracefully accepting constructive criticism
- Focusing on what is best for the community
- Showing empathy towards other community members

Examples of unacceptable behavior:
- Trolling, insulting/derogatory comments, personal attacks
- Public or private harassment
- Publishing others' private information without permission
- Other conduct which could reasonably be considered inappropriate

### Enforcement

Project maintainers have the right to remove, edit, or reject comments, commits, code, issues, and other contributions that do not align with this Code of Conduct.

## Development Setup

### Prerequisites
- Git
- Text editor (VS Code, Vim, etc.)
- Basic knowledge of JSON, Markdown

### Getting Started
```bash
# Fork and clone
git clone https://github.com/YOUR_USERNAME/AI-Discovery-Protocol.git
cd AI-Discovery-Protocol

# Create a branch
git checkout -b feature/your-feature

# Make changes
# ... edit files ...

# Commit
git add .
git commit -m "feat: add feature description"

# Push
git push origin feature/your-feature

# Open PR on GitHub
```
